# 网上书店系统软件需求规格说明书

## 一、文档说明

本文档是在实验六阶段 SRS 基础上，结合实验七对 SRS 模板、静态建模和角色需求分析的讨论后整理的版本。

## 二、范围

### 1. 项目标识

- 项目名称：网上书店系统
- 文档名称：软件需求规格说明书
- 文档阶段：课程项目推进中的阶段性初稿

### 2. 系统概述

网上书店系统是一个面向课程设计的 Java Web 项目，主要用于模拟在线书店的前台浏览购书与后台管理功能。

系统的核心目标包括：

- 支持普通用户浏览和购买图书
- 支持管理员对图书、订单、用户及网站内容进行维护
- 通过文档和模型方式逐步规范项目开发过程

### 3. 文档用途

本说明书主要用于：

- 统一组内对系统需求的理解
- 为后续设计文档和编码实现提供依据
- 为静态建模、动态建模和测试准备提供输入

## 三、总体描述

### 1. 产品透视

本系统为典型的 B/S 结构 Web 应用，采用浏览器访问方式，主要由前台业务和后台管理两部分组成。

系统整体采用如下技术结构：

- 表现层：JSP、HTML、CSS、JavaScript
- 控制层：Servlet
- 数据访问层：DAO
- 数据存储层：MySQL

### 2. 用户角色

| 角色     | 描述                                                       |
| -------- | ---------------------------------------------------------- |
| 游客     | 浏览图书、搜索图书、查看详情、查看网站内容、注册           |
| 注册用户 | 登录、加入购物车、提交订单、查看订单、确认收货             |
| 管理员   | 登录后台，维护图书、分类、用户、订单、公告、留言、网站信息 |

### 3. 运行环境

- 操作系统：Windows
- JDK：课程环境要求版本
- Web 容器：Tomcat
- 数据库：MySQL
- 浏览器：Chrome、Edge 等现代浏览器

### 4. 约束条件

- 本项目为课程设计项目，不接入真实支付系统
- 当前项目仍会随着实验推进继续修订需求
- 项目采用传统 Java Web 方式实现，不涉及微服务或分布式架构

## 四、需求概述

### 1. 系统目标

网上书店系统需要满足以下基本目标：

1. 为用户提供基础的图书查询和购买能力。
2. 为管理员提供后台维护能力。
3. 为课程实验提供可分析、可建模、可实现的完整系统对象。

### 2. 关键点

当前阶段小组认为本系统的关键点包括：

- 前后台功能边界要清楚
- 图书、订单和用户数据关系要明确
- 购物车与订单流程要完整
- 文档内容应能够支持后续建模和设计

## 五、功能需求

### FR-01 首页展示

系统应支持在首页展示推荐图书、新书上架以及必要的网站信息入口。

### FR-02 图书分类浏览

系统应支持用户查看图书分类，并按分类浏览图书列表。

### FR-03 图书搜索

系统应支持用户通过关键字搜索图书。

### FR-04 图书详情查看

系统应支持查看单本图书的详细信息，包括：

- 书名
- 作者
- 出版社
- 价格
- 简介
- 图片
- 分类信息

### FR-05 用户注册

系统应支持新用户注册。

基本要求：

- 用户名唯一
- 注册信息完整
- 注册后可用于登录

### FR-06 用户登录与退出

系统应支持普通用户登录和退出。

### FR-07 验证码校验

系统应支持登录或相关关键操作中的验证码校验。

### FR-08 购物车管理

系统应支持已登录用户进行购物车操作，包括：

- 加入购物车
- 修改数量
- 删除商品
- 清空购物车

### FR-09 提交订单

系统应支持从购物车生成订单。

前置条件：

- 用户已登录
- 购物车中存在商品

### FR-10 查看订单

系统应支持用户查看自己的订单信息。

### FR-11 确认收货

系统应支持用户对已发货订单进行确认收货。

### FR-12 管理员登录

系统应支持管理员登录后台管理系统。

### FR-13 用户管理

系统应支持管理员管理普通用户，主要包括：

- 查看用户
- 查询用户
- 修改用户
- 删除用户

### FR-14 图书分类管理

系统应支持管理员维护图书分类。

### FR-15 图书管理

系统应支持管理员维护图书信息，包括：

- 新增图书
- 修改图书
- 删除图书
- 查询图书
- 查看图书详情

### FR-16 订单管理

系统应支持管理员维护订单，包括：

- 查看订单
- 查看订单详情
- 发货处理
- 删除订单

### FR-17 网站内容维护

系统应支持管理员维护以下网站内容：

- 网站简介
- 公告
- 留言
- 留言回复或删除

## 六、外部接口需求

### 1. 用户接口

前台应提供：

- 首页
- 图书列表页
- 图书详情页
- 注册登录页
- 购物车页
- 我的订单页
- 公告与留言页

后台应提供：

- 管理员登录页
- 用户管理页
- 图书管理页
- 分类管理页
- 订单管理页
- 网站维护页

### 2. 软件接口

系统依赖以下外部软件环境：

- Tomcat
- MySQL
- JDBC

### 3. 数据接口

系统当前主要使用的数据表包括：

- `s_admin`
- `s_user`
- `s_catalog`
- `s_book`
- `s_uploadimg`
- `s_order`
- `s_orderitem`
- `s_siteinfo`
- `s_notice`
- `s_message`

## 七、数据需求

### 1. 核心实体

当前系统的核心实体包括：

- 管理员
- 用户
- 图书
- 图书分类
- 图书图片
- 订单
- 订单项
- 网站简介
- 公告
- 留言

### 2. 核心关系

- 一个用户对应多个订单
- 一个订单包含多个订单项
- 一个订单项对应一本图书
- 一本图书属于一个分类
- 一本图书关联一张图片

### 3. 主要数据规则

- 用户名不能重复
- 购物车中图书数量必须大于 0
- 订单金额应与订单项金额总和一致
- 图书应与分类建立明确对应关系

## 八、非功能需求

### 1. 可用性

- 页面结构应清晰
- 关键操作应有反馈提示
- 用户能较容易理解系统流程

### 2. 性能

- 常见查询功能应在可接受时间内完成
- 图书列表和订单列表应支持分页

### 3. 安全性

- 后台页面应限制未登录访问
- 用户只能查看自己的订单
- 非法输入不应导致系统直接崩溃

### 4. 可维护性

- 文档、模型和代码需要同步更新
- 模块职责要尽量清晰
- 命名方式应保持一致

### 5. 可扩展性

系统后续应能够继续补充新的模块或优化现有结构，例如：

- 增加库存管理
- 增强安全控制
- 补充更规范的业务层
