# 网上书店系统软件需求规格说明书

## 一、文档说明

本文档是在实验六阶段整理的网上书店系统软件需求规格说明书，用于描述当前阶段已经明确的软件需求，为后续设计、实现、联调和测试提供统一依据。

本说明书属于课程项目推进中的阶段性文档，后续若项目需求继续调整，应同步更新。

## 二、范围

### 1. 项目标识

- 项目名称：网上书店系统
- 文档名称：软件需求规格说明书（SRS）

### 2. 系统概述

网上书店系统是一个基于 Java Web 的课程设计项目，面向普通用户和管理员两类主要对象。系统目标是实现基础的图书浏览、会员购书和后台管理功能，并通过文档方式规范项目开发过程。

### 3. 文档用途

本说明书主要用于：

- 统一小组对系统需求的理解
- 作为后续设计与编码的依据
- 作为测试和验收时的参考基础

## 三、总体描述

### 1. 开发目标

本系统拟实现一个基本可运行的网上书店，支持前台用户浏览和购书，同时支持后台管理员进行图书和网站内容管理。

### 2. 运行环境

- 开发语言：Java
- Web 技术：Servlet、JSP、JavaScript、jQuery
- 数据库：MySQL
- 服务器：Tomcat
- 编码方式：UTF-8

### 3. 用户特点

系统主要面向以下用户：

- 游客：查看图书和网站信息
- 注册用户：登录后选购图书、提交订单、查看订单
- 管理员：登录后台管理用户、图书、订单和网站内容

### 4. 当前约束条件

- 项目属于课程设计，不涉及真实支付系统
- 开发周期有限，需要按实验进度逐步实现
- 文档会随着实验推进不断修订

## 四、功能需求

### FR-01 用户注册

系统应支持新用户注册。

注册信息至少包括：

- 用户名
- 密码
- 姓名
- 联系方式
- 地址

基本要求：

- 用户名不能重复
- 必填项不能为空
- 注册成功后可以用于登录

### FR-02 用户登录与退出

系统应支持普通用户登录和退出。

基本要求：

- 用户输入正确用户名和密码后可登录
- 登录状态应保存在会话中
- 用户可主动退出系统

### FR-03 图书分类浏览

系统应支持按图书分类浏览图书。

基本要求：

- 能展示图书分类列表
- 用户点击分类后可查看该分类下的图书
- 图书列表支持分页显示

### FR-04 图书搜索

系统应支持按关键字搜索图书。

基本要求：

- 用户输入书名关键字后可得到匹配结果
- 搜索结果支持分页显示

### FR-05 图书详情查看

系统应支持查看单本图书的详细信息。

展示内容至少包括：

- 图书名称
- 作者
- 出版社
- 价格
- 图片
- 简介

### FR-06 购物车管理

系统应支持用户对购物车进行管理。

主要功能包括：

- 加入购物车
- 修改图书数量
- 删除购物项
- 清空购物车
- 自动计算总数量和总金额

### FR-07 提交订单

系统应支持已登录用户提交订单。

基本要求：

- 用户必须先登录
- 购物车不能为空
- 提交后生成订单及订单项记录
- 提交成功后购物车应清空

### FR-08 我的订单

系统应支持用户查看自己的订单。

基本要求：

- 只能查看当前登录用户自己的订单
- 订单信息中应能看到金额和状态

### FR-09 确认收货

系统应支持用户对已发货订单执行确认收货操作。

### FR-10 管理员登录

系统应支持管理员登录后台。

基本要求：

- 账号密码验证正确后进入后台
- 未登录状态下不能直接访问后台管理页面

### FR-11 图书管理

系统应支持管理员对图书进行管理。

主要功能包括：

- 新增图书
- 修改图书
- 删除图书
- 查询图书
- 查看图书详情

### FR-12 分类管理

系统应支持管理员管理图书分类。

主要功能包括：

- 新增分类
- 删除分类
- 查询分类

### FR-13 用户管理

系统应支持管理员管理普通用户。

主要功能包括：

- 查看用户列表
- 查询用户
- 修改用户信息
- 删除用户

### FR-14 订单管理

系统应支持管理员管理订单。

主要功能包括：

- 查看订单列表
- 查看订单详情
- 删除订单
- 发货处理

### FR-15 网站内容维护

系统应支持管理员维护网站内容。

主要包括：

- 网站简介
- 公告
- 留言
- 留言回复或删除

## 五、外部接口需求

### 1. 用户接口

系统应提供前台与后台两个主要界面：

- 前台用于图书浏览、注册登录、购物和订单查看
- 后台用于图书、用户、订单和网站内容管理

### 2. 软件接口

系统依赖以下软件接口：

- Tomcat：用于运行 Web 项目
- MySQL：用于存储系统数据
- JDBC：用于进行数据库访问

### 3. 通信接口

本项目为本地或局域网环境下的传统 Web 应用，浏览器通过 HTTP 访问系统。

## 六、内部数据需求

### 1. 核心数据对象

当前系统涉及的核心数据对象包括：

- 管理员
- 用户
- 图书分类
- 图书
- 图书图片
- 订单
- 订单项
- 网站信息
- 公告
- 留言

### 2. 主要数据关系

- 一个用户可以对应多个订单
- 一个订单可以包含多个订单项
- 一个订单项对应一本图书
- 一本图书属于一个分类
- 一本图书对应一张图片

### 3. 数据基本规则

- 用户名应唯一
- 分类名称应尽量唯一
- 图书数量必须为正数
- 订单金额应与订单项金额总和一致

## 七、环境与资源需求

### 1. 硬件需求

对于课程项目，普通个人计算机即可满足开发和运行要求。

### 2. 软件需求

- JDK 8
- Tomcat 9
- MySQL
- 浏览器
- 开发工具IDEA

### 3. 通信需求

系统在浏览器和服务器之间通过 HTTP 完成请求和响应。

## 八、软件质量要求

### 1. 功能性

系统应覆盖前台浏览购书和后台管理两大类核心业务。

### 2. 可用性

页面结构应清晰，关键操作应有明确提示。

### 3. 可靠性

订单、购物车、登录等主要流程应尽量保持逻辑完整。

### 4. 可维护性

系统应保持基本分层结构，需求文档应可持续更新。

### 5. 可测试性

主要业务功能应可以通过页面操作或接口调用进行验证。

## 九、设计与实现约束

- 本项目为课程实验系统，不追求企业级完整架构
- 当前采用传统 `Servlet + JSP + DAO` 结构
- 不包含真实支付功能
- 不要求分布式部署

## 十、验收与合格性要求

本阶段可通过以下方式验证需求是否基本满足：

- 演示：直接通过页面操作展示功能
- 测试：对注册、登录、浏览、下单、后台管理等进行基本功能测试
- 审查：检查代码、数据库和文档是否与需求一致

## 十一、需求可追踪性

| 需求类别                    | 对应模块       |
| --------------------------- | -------------- |
| 用户注册/登录               | 用户模块       |
| 图书浏览/搜索/详情          | 图书模块       |
| 购物车/订单                 | 购物与订单模块 |
| 后台图书/分类/用户/订单管理 | 后台管理模块   |
| 简介/公告/留言              | 网站维护模块   |

## 十二、尚未完全解决的问题

当前阶段仍有一些问题需要在后续实验中继续完善：

- 更细致的接口需求还不够完整
- 非功能需求还可以继续细化
- 一些边界条件和异常处理场景需要在测试阶段进一步补充
- 文档仍需随着项目实现继续修订
