# 网上书店系统软件需求规格说明书（SRS）

## 1. 引言

### 1.1 目的

本文档用于说明“网上书店系统”的功能需求、数据需求、接口需求和非功能需求，为后续设计、编码、测试和课程验收提供统一基线。

### 1.2 范围

系统面向两个主要使用场景：

- 前台用户在线浏览图书、注册登录、购物和查询订单
- 后台管理员维护图书、分类、用户、管理员和订单

### 1.3 术语

| 术语 | 说明 |
| --- | --- |
| 用户 | 书店前台注册用户 |
| 管理员 | 后台运维与数据维护人员 |
| 订单状态 1 | 待处理 |
| 订单状态 2 | 已发货 |
| 订单状态 3 | 已完成/已收货 |
| 购物车 | 用户会话中的待购图书集合 |

## 2. 总体描述

### 2.1 产品透视

系统是一个基于浏览器访问的 B/S 结构应用，采用：

- 表现层：JSP、HTML、CSS、JavaScript、Bootstrap、jQuery
- 控制层：Servlet
- 数据访问层：DAO
- 数据层：MySQL

### 2.2 用户角色

| 角色 | 描述 |
| --- | --- |
| 游客 | 可浏览、搜索、查看图书详情、注册 |
| 注册用户 | 可登录、加入购物车、提交订单、查看订单、确认收货 |
| 管理员 | 可登录后台并维护管理员、用户、图书、分类、订单 |

### 2.3 运行环境

- 操作系统：Windows
- JDK：25
- Web 容器：Tomcat 9
- 数据库：MySQL 8
- 浏览器：Chrome/Edge 等现代浏览器

### 2.4 约束

- 当前项目为课程实验系统，不包含真实支付网关
- 用户密码当前按示例项目方式存储，后续可升级为加密存储
- 系统以单体 Web 应用形式部署，不涉及分布式拆分

## 3. 功能需求

### FR-01 前台首页展示

系统应支持在首页展示推荐图书

- 输入：无
- 处理：从图书数据源读取推荐
- 输出：图书列表 JSON/页面展示

### FR-02 分类浏览

系统应支持按图书分类展示图书列表。

- 用户可查看分类列表
- 用户可按分类分页浏览图书

### FR-03 图书搜索

系统应支持按图书名称关键字检索图书。

- 输入：图书名称关键字
- 输出：匹配图书列表

### FR-04 图书详情

系统应支持查看单本图书详情。

- 展示内容至少包括书名、作者、出版社、价格、描述、图片和分类信息

### FR-05 用户注册

系统应支持新用户注册。

- 必填信息：用户名、密码、姓名、性别、年龄、电话、地址
- 用户名必须唯一
- 注册成功后用户状态默认为启用

### FR-06 用户登录与退出

系统应支持用户登录、Ajax 登录状态检查和退出登录。

- 输入：用户名、密码
- 处理：校验用户名密码与启用状态
- 输出：登录成功进入首页；失败返回提示信息

### FR-07 验证码校验

系统应支持用户登录/结算过程中的验证码生成与校验。

### FR-08 购物车管理

系统应支持用户在会话中维护购物车：

- 加入购物车
- 修改商品数量
- 删除购物项
- 清空购物车
- 实时计算总数量与总金额

### FR-09 提交订单

系统应支持登录用户从购物车生成订单。

前置条件：

- 用户已登录
- 购物车非空

处理规则：

- 生成唯一订单号
- 新建订单记录，初始状态为“待处理”
- 为购物车中的每个商品生成订单项
- 提交成功后清空购物车

### FR-10 查看我的订单

系统应支持用户按分页查看自己的订单。

- 用户只能查看自己的订单
- 应展示订单项、图书信息、金额和状态

### FR-11 确认收货

系统应支持用户对已发货订单执行确认收货。

- 只有状态为“已发货”的订单允许确认
- 确认后订单状态变为“已完成”

### FR-12 管理员登录

系统应支持管理员登录后台管理界面。

- 输入：管理员用户名、密码
- 成功后建立后台登录会话

### FR-13 管理员管理

系统应支持管理员维护管理员账号：

- 分页查看
- 新增
- 修改
- 删除
- 批量删除
- 用户名唯一校验

### FR-14 用户管理

系统应支持管理员维护普通用户：

- 分页查看
- 条件查询
- 查看详情
- 新增
- 修改
- 删除
- 批量删除
- 用户启用/禁用

### FR-15 分类管理

系统应支持管理员维护图书分类：

- 分页查看
- 新增
- 删除
- 批量删除
- 分类名称唯一校验

### FR-16 图书管理

系统应支持管理员维护图书：

- 分页查看
- 条件查询
- 查看详情
- 新增图书
- 修改图书信息
- 修改图书图片
- 删除图书
- 批量删除图书

图书新增时应同时满足：

- 必须选择分类
- 必须上传一张合法图片
- 图书名不重复

### FR-17 订单管理

系统应支持管理员维护订单：

- 分页查看全部订单
- 按订单号查询
- 查看订单详情
- 删除订单
- 查看待处理订单
- 发货处理

### FR-18 在线访问辅助信息

系统可记录访问计数、在线用户等辅助信息，用于后台展示和系统运行观察。

## 4. 外部接口需求

### 4.1 用户界面

- 前台页面应提供首页、图书列表、图书详情、注册登录、购物车、订单列表
- 后台页面应提供管理员、用户、图书、分类、订单管理界面
- 关键操作应有成功/失败提示

### 4.2 软件接口

| 接口对象 | 说明 |
| --- | --- |
| Tomcat | 承载 JSP/Servlet 运行 |
| MySQL | 存储用户、图书、订单等数据 |
| JDBC | 由 `DbUtil` 负责建立数据库连接 |

### 4.3 数据接口

核心数据表包括：

- `s_admin`
- `s_user`
- `s_catalog`
- `s_book`
- `s_uploadimg`
- `s_order`
- `s_orderitem`
- 视图：`view_book`、`view_order`

## 5. 数据需求

### 5.1 核心实体

| 实体 | 关键字段 |
| --- | --- |
| Admin | id, userName, passWord, name |
| User | userId, userName, passWord, name, enabled |
| Catalog | catalogId, catalogName |
| Book | bookId, bookName, catalogId, price, imgId |
| UpLoadImg | imgId, imgName, imgSrc, imgType |
| Order | orderId, orderNum, userId, money, orderStatus |
| OrderItem | itemId, orderId, bookId, quantity |
| Cart | 会话内 Map<bookId, CartItem> |

### 5.2 数据关系

- 一个用户可拥有多个订单
- 一个订单可拥有多个订单项
- 一个订单项对应一本图书
- 一本图书属于一个分类
- 一本图书关联一张图片

### 5.3 数据规则

- 用户名必须唯一
- 分类名必须唯一
- 图书名应唯一
- 订单金额应等于所有订单项金额之和
- 购物车中商品数量必须大于 0

## 6. 非功能需求

### 6.1 性能

- 普通查询操作应在 3 秒内完成
- 首页、图书列表、订单列表应支持分页

### 6.2 安全性

- 后台路径应通过过滤器限制未登录访问
- 用户只能查看自己的订单
- 非法参数输入不应导致系统崩溃

### 6.3 可用性

- 前后台页面应有清晰导航
- 增删改查操作应返回明确提示
- 关键输入项应做表单校验

### 6.4 可靠性

- 订单提交过程应保证“订单 + 订单项”逻辑完整
- 图书删除时应同步处理图片记录与物理文件

### 6.5 可维护性

- 控制器、DAO、实体对象分层明确
- URL 与动作命名应保持一致
- 需求文档、模型与代码应同步更新

## 7. 验收标准

| 编号 | 验收项 | 验收标准 |
| --- | --- | --- |
| A1 | 管理员登录 | 输入正确账号密码可进入后台 |
| A2 | 用户注册登录 | 新用户可注册，已启用用户可登录 |
| A3 | 图书浏览 | 首页、分类、详情、搜索均可用 |
| A4 | 购物车 | 可新增、修改、删除、清空并更新金额 |
| A5 | 提交订单 | 下单后生成订单和订单项，购物车清空 |
| A6 | 我的订单 | 用户仅能看到自己的订单 |
| A7 | 后台图书管理 | 可完成图书增删改查与图片更新 |
| A8 | 后台订单管理 | 管理员可查看、发货、删除订单 |

## 9. 项目边界说明

本系统为课程实验项目，当前不包含：

- 真实在线支付
- 密码加密与完整安全审计
- 分布式部署与高并发支撑
- 运营数据分析、推荐算法、库存系统
